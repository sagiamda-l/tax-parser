# 📑 Tax-Parser: 통합 연말정산 및 지출 내역 분석 시스템

본 프로젝트는 개인의 다양한 지출 내역(카드, 기부금, 교육비, 은행 입출금 등)에서 종합소득세 신고 시 필요경비(일반관리비)로 인정받을 수 있는 항목을 자동으로 식별하고 분류하기 위한 풀스택 관리 도구입니다.

---

## 파서 엔진 (Parser Engine) 상세

본 시스템은 국내 주요 카드사의 이용내역 엑셀 파일을 자동 감지하여 통합 파싱합니다.

### **지원 카드사 및 문서**

- **신한카드**: 이용내역(일반/법인)
- **KB국민카드**: 이용내역 (국내이용금액 컬럼 대응 완료)
- **삼성카드**: 이용내역 상세
- **NH농협카드**: 이용내역서 (이용금액(원) 대응 완료)
- **토스카드**: 결제 내역 (결제처/결제일시 대응 완료)
- **연말정산 간소화 자료 (PDF)**: 국세청 홈택스 발행 PDF의 의료비, 교육비, 기부금 내역 추출
- **원천징수 영수증 (PDF)**: 소득자 정보 및 지급 내역 분석

### **파싱 로직 원칙**

1. **헤더 자동 탐색**: 파일 상단에 불필요한 메타데이터가 있어도 키워드('금액', '가맹점')를 기준으로 데이터 시작점을 찾습니다.
2. **데이터 정제**: 금액에 포함된 특수문자(`,`, `원`)를 제거하고 순수 수치 데이터로 변환합니다.
3. **사용자 매핑**: 엑셀 내 '이용자' 컬럼이 없을 경우 업로드 주체인 '이성렬'로 기본 매핑합니다.

### **지능형 분류 (Smart Tagging)**

- 파싱 단계에서 가맹점명(vendor) 키워드 분석을 통해 **9가지 세무 항목**을 1차 제안합니다.
- 예: '주유소' 포함 시 -> [차량유지비], '택시' 포함 시 -> [여비교통비]

## 1. 프로젝트 목적 및 분류 체계

본 시스템의 핵심은 흩어져 있는 지출 데이터를 수집하여 아래의 9가지 세무 비용 항목으로 분류하고 검토하는 것입니다.

핵심 비용 분류 항목
기업업무추진비: 접대비 및 업무 관련 교제비
기부금: 지정 및 법정 기부금 내역
차량유지비: 유류비, 수리비, 통행료 등
지급수수료: 송금 수수료 및 각종 서비스 이용료
소모품비: 사무용품 및 소모성 자재 구입비
운반비: 택배 및 화물 운송 비용
광고선전비: 마케팅 및 홍보 관련 지출
여비교통비: 출장비, 대중교통 이용료
기타: 위 항목에 속하지 않는 일반 관리 비용

## 🛠 2. 기술적 원칙 (Ground Rules)

### A. 데이터 무결성 및 PK 규칙

- 모든 파일 레코드는 추적 가능성을 위해 고유한 PK를 가집니다.
  - PK 생성 수식: $$PK = \text{CurrentTimestamp}(17\text{ digits}) + \text{Random}(5\text{ digits})$$
    예: 20260510182218123 (시간) + 45678 (난수) = 2026051018221812345678
  - Cascade 삭제: 상위 파일(UploadFileRecord) 삭제 시, 해당 파일에서 파싱된 모든 내역(Card/Document Records)은 데이터베이스에서 즉시 삭제됩니다.
- 업로드 및 중복 제어 워크플로우
  - 중복 발생 시: 클라이언트에게 `409 Conflict` 반환 -> 팝업을 통해 '덮어쓰기(Overwrite)' 확인.
  - 신규 파일: 즉시 파싱 및 저장.
- **이력 저장**: PK, 파일명, 업로드 시간, 이용자, 문서종류, 대상년도 정보를 기록합니다.

### 2. 상세 내역 그리드 (Data Grid)

- 파일명 체크: 동일한 파일명이 존재할 경우 서버는 409 Conflict 에러를 반환합니다.
- 사용자 선택:
  - 취소: 업로드를 중단합니다.
  - 확인(덮어쓰기): 기존 파일 및 연관된 파싱 데이터를 모두 삭제한 후 신규 데이터를 등록합니다.

## 🖥 3. UI/UX 요구사항

- A. 상세 내역 그리드 (Grid)
  - 표시 정보: 문서명, 이용자명, 결제 일자, 금액, 가맹점, 태그(비용 분류).
  - 태그 시스템: 9가지 비용 항목별로 고유 아이콘과 배경색을 적용하여 시각적 인지력을 높입니다.
  - 일괄 저장: 그리드에서 수정한 태그 정보를 [태그변경사항 저장] 버튼 클릭 시 한 번에 DB에 반영합니다.
- B. 필터링 시스템
  - 대상 년도: 당해년도 데이터를 기본으로 조회합니다.

  - 동적 콤보박스: 현재 화면에 로드된 데이터를 기반으로 이용자명과 문서명 필터를 생성합니다.

## 📂 4. 시스템 구조 (Architecture)

- Frontend: React (Recharts 시각화, Axios 통신)
- Backend: FastAPI (Python 3.11), SQLAlchemy (ORM)
- Database: SQLite (폐쇄망 운영 최적화)
- Deployment: Docker-compose 기반 컨테이너화
